# G0DM0D3 — Условия использования

**Последнее обновление:** 2026-02-17

---

## 1. Принятие условий

Используя G0DM0D3 — через веб-интерфейс, сервер API или собственный развёрнутый экземпляр — вы соглашаетесь с этими условиями. Если вы не согласны, не используйте программное обеспечение.

---

## 2. Что такое G0DM0D3

G0DM0D3 — это открытый исследовательский инструмент для AI-безопасности, red teaming и исследований когниции. Он предоставляется **как есть** для экспериментов и изучения. Это не коммерческий продукт, и за ним не стоит компания — только открытый проект и его участники.

---

## 3. Сбор данных

Этот раздел — самый важный. Прочтите его внимательно.

G0DM0D3 собирает данные в **трёх уровнях**. Первые два работают постоянно и собирают **только метаданные** — никогда не ваши сообщения, промпты, ответы, ключи API или любую персонально идентифицируемую информацию. Третий уровень включается только по выбору и может включать содержимое разговора.

### Обзор

| | Уровень 1: ZDR Metadata | Уровень 2: Telemetry Beacon | Уровень 3: Сбор данных |
|---|---|---|---|
| **Где работает** | Сервер API | Веб-фронтенд (браузер) | Сервер API |
| **Всегда включено?** | Да | Да | **Нет — только по выбору** |
| **Собирает содержимое сообщений?** | Никогда | Никогда | **Да, если вы включили** |
| **Собирает ПИИ?** | Никогда | Никогда | Никогда |
| **Собирает ключи API?** | Никогда | Никогда | Никогда |
| **Исходный код** | `api/lib/metadata.ts` | `src/lib/telemetry.ts` | `api/lib/dataset.ts` |

---

### Уровень 1: Трекер ZDR Metadata (Всегда включено — сервер API)

Это выполняется при каждом API-запросе. Он фиксирует, как система работала, а не что вы сказали.

#### Что собирается

| Пункт данных | Описание |
|---|---|
| Timestamp | Время запроса |
| Endpoint | Какой API-эндпоинт вызван |
| Mode | Standard или ULTRAPLINIAN |
| Tier | fast, standard или full (только ULTRAPLINIAN) |
| Stream | Используется ли потоковая передача |
| Pipeline flags | Включены ли godmode, autotune, parseltongue и STM |
| STM modules | Какие STM-модули активны |
| Strategy | Какая стратегия маршрутизации используется |
| AutoTune context | Обнаруженный тип контекста (code/creative/analytical/conversational/chaotic) и уровень уверенности |
| Models queried | Сколько моделей было опрошено (только количество) |
| Per-model results | Имя модели, счёт, длительность в мс, успех/ошибка, длина содержимого (только символы), категория ошибки |
| Winner metadata | Имя победившей модели, счёт, длительность, длина содержимого (символы) |
| Total duration | Общее время запроса в мс |
| Response length | Длина ответа в символах (не сам ответ) |
| Liquid Response metadata | Количество смен лидера, время до первого ответа |
| Error types | Категории: `timeout`, `rate_limit`, `auth`, `empty`, `early_exit`, `model_error`, `unknown` |

#### Что НИКОГДА не собирается

| Исключённые данные | Почему |
|---|---|
| Содержимое сообщений | Ваши промпты и разговоры ваши |
| Системные промпты | Никогда не сохраняются |
| Текст ответа | Только длина в символах, не сам ответ |
| Ключи API или токены | Никогда не покидают клиент |
| IP-адреса | Не фиксируются |
| Любые ПИИ | Не фиксируются, точка |
| Сырые сообщения об ошибках | Только категория ошибки, не текст |

**Хранение:** кольцевой буфер в памяти (максимум 50 000 событий). Когда буфер достигает 80% заполнения, он автоматически публикуется на HuggingFace в формате JSONL и очищается. Старые события удаляются при заполнении.

### Уровень 2: Клиентская телеметрия (Всегда включено — веб-фронтенд)

Это выполняется в браузере после каждого запроса к LLM. Он отправляет структурные метаданные на прокси-эндпоинт Cloudflare Pages. Прокси защищает данные, удаляя ПИИ перед отправкой на HuggingFace.

#### Что собирается

| Пункт данных | Описание |
|---|---|
| Event type | Тип события (например, `chat_completion`) |
| Timestamp | Время события |
| Session ID | Случайный временный идентификатор сессии — не связан с пользователем |
| Mode | Standard или ULTRAPLINIAN |
| Model name | Какая модель использовалась |
| Duration | Время запроса в мс |
| Response length | Длина ответа в символах |
| Success/failure | Успешен ли запрос |
| Pipeline config | Autotune включён/выключен, parseltongue включён/выключен, STM-модули, стратегия, godmode включён/выключен |
| AutoTune metadata | Обнаруженный тип контекста и уверенность |
| Parseltongue metadata | Количество найденных триггеров (только число), техника, интенсивность |
| ULTRAPLINIAN metadata | Уровень, количество опрошенных моделей, количество успешных моделей, победитель, счёт победителя, общее время |

#### Что НИКОГДА не собирается

| Исключённые данные | Почему |
|---|---|
| Содержимое сообщений | Никогда не отправляется |
| Промпты или ответы | Не включаются в события |
| Ключи API или токены | Никогда не покидают localStorage |
| Любые ПИИ | Не собираются; прокси также очищает данные |

**Хранение:** события накапливаются в памяти браузера и отправляются каждые 30 секунд или каждые 20 событий (что наступит раньше) через POST на `/api/telemetry`. При закрытии страницы оставшиеся события отправляются через `navigator.sendBeacon`. Телеметрия отправляется без блокировки UI и не вызывает пользовательских ошибок.

---

### Уровень 3: Сбор данных по выбору (только при явном включении)

> **ПРЕДУПРЕЖДЕНИЕ: этот уровень собирает полное содержимое разговоров (промпты и ответы) и публикует его в ПУБЛИЧНЫЙ набор данных HuggingFace, который любой может скачать, форкнуть и использовать как угодно. Включайте только если вы понимаете и принимаете это.**

Этот уровень **включается только**, когда вы явно активируете «Генерацию данных» в Настройках → Конфиденциальность или устанавливаете `contribute_to_dataset: true` в запросе API. Он предназначен для создания открытого исследовательского набора данных для изучения реакций моделей. Если вы не включили его, эти данные не собираются.

#### Что собирается (только при включении)

| Пункт данных | Описание |
|---|---|
| Сообщения отправленные и полученные | **Фактическое содержимое разговора** — ваши промпты и ответы модели. **Эти данные ПУБЛИЧНЫ.** |
| Модель и endpoint | Какая модель и API-эндпоинт использовались |
| Режим | Standard или ULTRAPLINIAN |
| Детали AutoTune | Стратегия, обнаруженный контекст, уверенность, полный набор параметров, рассуждения |
| Детали Parseltongue | Найденные триггерные слова (сам текст, не только число), используемая техника, количество преобразований |
| Детали STM | Какие модули применялись |
| Детали ULTRAPLINIAN | Уровень, полный список опрошенных моделей, победитель, все оценки/длительности, общее время |
| Обратная связь | Оценки пользователя (палец вверх/вниз) и эвристики (длина ответа, повторяемость, разнообразие слов) |

#### Автоматическое удаление ПИИ

Все записи набора данных проходят автоматический PII scrubber перед хранением. Скраббер обрабатывает:

- Адреса электронной почты
- Номера телефонов (US и международные форматы)
- Социальные номера (SSN)
- Номера кредитных карт
- IPv4 и IPv6 адреса
- Ключи API и bearer-токены (типичные шаблоны `sk-`, `pk-`, `AKIA`)

**Автоматическое удаление — это лучший доступный механизм, но оно НЕ гарантирует обнаружение всего.** Оно не поймает имена, физические адреса или нестандартные форматы. **Вы несёте ответственность за то, чтобы не включать чувствительные личные данные при включенном режиме набора данных.**

#### Что НИКОГДА не следует включать при Dataset Mode

- Настоящие имена (ваши или чужие)
- Физические адреса или геолокацию
- Пароли, приватные ключи или учётные данные
- Финансовые номера счетов
- Медицинские записи или данные о здоровье
- Любую информацию, которая может идентифицировать реального человека

#### Что НИКОГДА не собирается (даже при включении)

| Исключённые данные | Почему |
|---|---|
| Ключи API | Никогда не хранится, никогда не передаётся на инфраструктуру G0DM0D3 |
| IP-адреса | Не фиксируются |
| Токены авторизации | Не фиксируются |
| Системные промпты | Исключены, чтобы не допустить утечку пользовательской конфигурации промптов |

**Хранение:** буфер в памяти (максимум 10 000 записей, FIFO). Автоматическая публикация на HuggingFace при заполнении буфера на 80%.

**Удаление:** вы можете удалить свои записи в любой момент через `DELETE /v1/dataset/:id`, пока данные остаются в памяти сервера. **После публикации на HuggingFace данные становятся публичными и могут сохраняться, форкаться или распространяться третьими лицами.** Запросы на удаление после публикации должны направляться администраторам репозитория HuggingFace, и полное удаление не гарантируется.

## 4. Хранение данных и публикация

### Хранение в памяти

Все собранные данные (метаданные, телеметрия и записи набора данных) сначала хранятся в памяти сервера. Ничего не записывается на диск на сервере G0DM0D3. Если сервер перезапускается, несохранённые данные теряются.

### Публикация на HuggingFace

При настройке (`HF_TOKEN` и `HF_DATASET_REPO` в переменных окружения) G0DM0D3 автоматически публикует собранные данные в репозиторий набора данных HuggingFace.

| Деталь | Значение |
|---|---|
| Репозиторий по умолчанию | `pliny-the-prompter/g0dm0d3` |
| Формат файла | JSONL (JSON Lines) |
| Путь метаданных | `metadata/batch_<timestamp>.jsonl` |
| Путь набора данных | `dataset/batch_<timestamp>.jsonl` |
| Триггеры сброса | буфер на 80% заполнения, таймер каждые 30 минут, или корректное завершение сервера |

Публично опубликованные данные на HuggingFace остаются доступными постоянно. После публикации они подпадают под условия HuggingFace.

Самостоятельные развёртывания могут отключить публикацию на HuggingFace, просто не задав `HF_TOKEN` и `HF_DATASET_REPO`. В этом случае данные остаются только в памяти и теряются при перезапуске.

### Срок хранения данных

- **В памяти:** данные хранятся до публикации на HuggingFace или удаления по FIFO при заполнении буфера.
- **На HuggingFace:** опубликованные данные сохраняются бессрочно, если только их не удалят вручную.

### Права на удаление

- **Уровень 3 (по выбору):** вы можете запросить удаление отдельных записей через `DELETE /v1/dataset/:id`, пока данные ещё находятся в памяти. После публикации запросы на удаление направляются администраторам репозитория HuggingFace.
- **Уровни 1 и 2 (метаданные/телеметрия):** поскольку эти данные не содержат ПИИ и содержат только агрегированные анонимные метаданные, отдельное удаление не предусмотрено.

## 5. Ключи API, учётные данные и история чата

Ваши ключи API (например, OpenRouter) — это:

- **Хранятся только в `localStorage` браузера.** Они никогда не покидают ваше устройство, за исключением отправки напрямую поставщику API (например, OpenRouter) в рамках запроса.
- **Никогда не передаются на серверы или инфраструктуру G0DM0D3.**
- **Никогда не логируются, не записываются и не включаются в телеметрию или метаданные.**

Вы ответственны за безопасность своих ключей API. Не публикуйте их.

### Самостоятельное хранение истории чата

Ваша история разговора хранится **исключительно в `localStorage` браузера**. У G0DM0D3 нет системы аккаунтов, нет облачного синхронизированного хранилища и нет серверной записи ваших разговоров.

**Это означает:**

- Очистка данных браузера **навсегда удаляет** историю чата. Восстановления нет.
- История не передаётся между браузерами, устройствами или профилями.
- Частный/инкогнито режим удаляет всё при закрытии окна.

**G0DM0D3 не может восстановить потерянные разговоры.** Ни одна копия истории чата не хранится на сервере. Вы сами ответственны за резервное копирование важных разговоров. Вы можете экспортировать данные вручную через инструменты разработчика браузера (`Application → Local Storage`) или копированием из UI.

## 6. Корпоративное использование

### Кому это применяется

Этот раздел относится к **корпоративным пользователям** — любому коммерческому юридическому лицу (или его дочерним/аффилированным структурам и подрядчикам), которое соответствует **любому** из следующих критериев:

- Годовой валовой доход более **10 миллионов долларов США**
- Более **100 сотрудников** (полный/неполный рабочий день или подрядчики)
- Торгуется на любой фондовой бирже
- Является дочерней компанией или подразделением организации, подпадающей под перечисленные критерии

Если вы — частное лицо, исследователь, студент, энтузиаст, небольшая команда, некоммерческая организация, образовательное учреждение, правительственная исследовательская лаборатория или компания, не соответствующая этим порогам — **этот раздел к вам не применяется. G0DM0D3 бесплатен для вас. Точка.**

### Что должны делать корпоративные пользователи

Корпоративные пользователи, которые используют хостинг-сервис G0DM0D3 (сервер API, веб-фронтенд проекта или любую инфраструктуру проекта) любым способом — включая внутренние инструменты, продукты для сотрудников, продукты для клиентов, оценку, бенчмаркинг или интеграцию — должны получить **лицензию на коммерческое использование** у поддерживающих проект.

Это относится к:

- **Прямому использованию** хостированного сервиса или API
- **Встраиванию** хостированного сервиса в корпоративные продукты, платформы или рабочие процессы
- **Использованию данных, собранных через хостированный сервис** (Уровень 1, 2 или 3) для коммерческих целей: обучение моделей, разработка продукта, конкурентный анализ или внутренние исследования, связанные с коммерческими продуктами

### Что получают корпоративные пользователи

Лицензия на коммерческое использование включает:

- Разрешённое использование хостированного сервиса G0DM0D3 для корпоративных нужд
- Канал приоритетной поддержки
- Гарантии SLA (по согласованию)
- Пользовательские условия обработки данных и хранения (при необходимости)
- Освобождение от стандартного процесса публикации данных (Уровни 1 и 2), если это согласовано контрактом

### На что это не распространяется

Чтобы было совершенно ясно, следующие случаи **всегда бесплатны, для всех, включая корпорации**:

- **Самостоятельный хостинг G0DM0D3 из исходников** по лицензии AGPL-3.0. Лицензия исходного кода отделена от этих сервисных условий. Если вы клонируете репозиторий, запускаете его на своих серверах и соблюдаете AGPL-3.0 (оставляете производные открытыми), вы ничего не должны.
- **Вклад в проект** через pull request, issue, документацию или поддержку сообщества.
- **Академические или некоммерческие исследования** с использованием хостированного сервиса независимо от размера организации.
- **Личное использование** сотрудниками корпораций в индивидуальном порядке, в свободное от работы время, с собственными ключами API.

### Стоимость

Стоимость лицензий на коммерческое использование обсуждается индивидуально. Корпоративным пользователям следует связаться с поддержкой проекта для обсуждения условий.

### Исполнение

Корпоративные пользователи, получающие доступ к хостированному сервису без лицензии на коммерческое использование, нарушают эти условия. Поддержка проекта оставляет за собой право:

- Отозвать доступ к хостированному сервису
- Потребовать разумную компенсацию за несанкционированное коммерческое использование
- Обнародовать нарушения (после 30-дневного уведомления и срока на исправление)

Мы не стремимся судиться с кем-либо. Лучше вести диалог. Но если миллиардная компания извлекает ценность из этого проекта и ничего не возвращает — ни кодом, ни финансированием, ни признанием — это не дух open source. Этот пункт создан, чтобы сохранить справедливость.

### Лицензирование данных для корпораций

Корпоративные пользователи, получающие доступ к опубликованному исследовательскому набору данных (на HuggingFace) для коммерческих целей — включая обучение моделей, дообучение, разработку продукта или конкурентную разведку — должны получить **лицензию на использование данных** у поддерживающих проект.

Некомерческое исследовательское использование опубликованного набора данных всегда бесплатно.

## 7. Ваши обязанности

Используя G0DM0D3, вы соглашаетесь с тем, что:

- **Вы несёте ответственность за результаты, сгенерированные через этот инструмент.** G0DM0D3 — это исследовательский интерфейс: сам по себе он не генерирует контент, он лишь направляет запросы к моделям ИИ через сторонние API.
- **Вы соблюдаете все применимые законы и правила** в вашей юрисдикции.
- **Вы соблюдаете условия использования базовых провайдеров ИИ** (например, OpenRouter и провайдеров моделей, доступных через него).
- **Вы не будете использовать G0DM0D3 для генерации контента, который является незаконным** в вашей юрисдикции.
